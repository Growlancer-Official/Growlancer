# Contributing to Growlancer

Thank you for your interest in contributing to Growlancer.

## Before You Start

Please create an Issue before working on any feature, enhancement, or bug fix.

Do not begin development without approval from the project owner.

---

## Contribution Process

1. Forking is not accepted for official development.
2. Clone the repository.
3. Create a new branch.
4. Complete the assigned task.
5. Test your changes.
6. Open a Pull Request.
7. Wait for review and approval.

---

## Coding Standards

- Write clean, readable code.
- Follow the existing project structure.
- Do not remove or modify unrelated code.
- Keep commits meaningful.
- Test before submitting.

---

## Pull Requests

Every Pull Request must include:

- Clear title
- Description of changes
- Screenshots (if UI changes)
- Related Issue Number

---

## Reporting Bugs

Please include:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots
- Browser / Device information

---

## Feature Requests

Before requesting a feature:

- Search existing Issues.
- Explain the problem.
- Describe the proposed solution.

---

## Confidentiality

This repository contains proprietary Growlancer source code.

Contributors, interns, employees, and collaborators must not copy, redistribute, publish, or reuse any part of this codebase outside official Growlancer projects.

Unauthorized distribution or commercial use is strictly prohibited.
